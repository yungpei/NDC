寫 WSL Ubuntu 裡的完整編譯與執行教學指令

1. 安裝 WSL (Windows Subsystem for Linux) 與 Ubuntu

在 Windows Powershell (管理員) 執行：wsl --install
如果已有 WSL，請確定安裝了 Ubuntu 20.04 或 22.04：wsl -l -v
若沒有安裝 Ubuntu，請從 Microsoft Store 安裝 Ubuntu。

2. 啟動 Ubuntu

3. 更新系統並安裝必要工具：

sudo apt update && sudo apt upgrade -y
sudo apt install -y build-essential cmake git

進入 NDC 專案資料夾
專案路徑是 E:\wu\NDC ， WSL 裡對應路徑是 /mnt/e/wu/NDC：cd /mnt/e/wu/NDC/data_utils

編譯每個需要的子資料夾
以 get_exact_sdf 為例：

cd get_exact_sdf
cmake .
make

編譯完成後，會在本資料夾出現執行檔（或在 bin 資料夾，視專案結構而定）。
對其他資料夾同樣操作，例如：

cd ../get_exact_intersections
cd get_exact_intersections
cmake .
make

cd ../get_exact_intersections_with_normals
cmake .
make

cd ../get_exact_sign_voxel
cmake .
make

cd ../get_fusion_sdf
cmake .
make

執行產生 sdf、binvox 等檔案的指令
假設你已把 .obj 檔案放到適當資料夾，依照 get_gt_LOD.py 的用法，你需要執行：

(base) cg@DESKTOP-QGNGKIV:/mnt/e/wu/NDC$ cd data_utils
(base) cg@DESKTOP-QGNGKIV:/mnt/e/wu/NDC/data_utils$ cd get_exact_intersections
(base) cg@DESKTOP-QGNGKIV:/mnt/e/wu/NDC/data_utils/get_exact_intersections_with_normals$ cd bin

./VOXGen model.obj 1024 0
./SDFGen model.obj 512 0
./IntersectionXYZpn model.obj 512 0

./VOXGen model.obj 512 0
./SDFGen model.obj 512 0
./IntersectionXYZpn model.obj 512 0

./SDFcarve model.obj 512 0
./IntersectionXYZ model.obj 512 0


執行 Python 腳本
確定你在 WSL Ubuntu 裡有 Python 環境（建議用 Anaconda 或直接用系統 Python）
安裝必要套件：

sudo apt install python3 python3-pip
pip3 install numpy

進入你的 get_groundtruth_NDC 目錄： cd /mnt/e/wu/NDC/data_preprocessing/get_groundtruth_NDC
執行：python3 get_gt_LOD.py
